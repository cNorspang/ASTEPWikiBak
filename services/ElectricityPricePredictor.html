Electricity Price Predictor

The microservice was created in 2021 by cs-21-sw-5-08. In order to predict electrictricity prices using the microservice, the user will have to upload a CSV file with data formatted in the correct way as seen in here LINK. If a CSV file is not uploaded, the user can predict electricity prices by using the default dataset.
Users can choose between multiple prediction methods such as SARIMAX, ARIMA.

 

Current Status

The microservice is fully functional, but has room for improvement. 

The microservice can predict electricity prices within a user specified interval, however the recommended prediction is with 24 hours, as predictions later than that will be predicted using predicted data. This could be something to look at in future semesters as the output looks the same after 24 hours. 

Furthermore even more exogenous variables could be added in order to get a more precise result. 
Another improvement could be to implement more models or improve the excisting once. 

 

 

Inner Workings

The microservice was created using Python, to host the back-end the Flask framework was chosen. Furthermore some python packages to help predict the electricity, primarily the statsmodels library (can be found here) has been used, as it was a widely used and supported time-series analysis package. The SARIMAX class was the driving factor of the microservice as it allowed for predictions, while also accounting for seasonality.

The repository for the microservice can be found at: Electricity Price Prediction GitLab.

The SQLAlchemy ORM was used to establish connection to the aSTEP database.  

 

History
